## Обзор системы

**SKFET Notification System** — это масштабируемая платформа для оповещения и коммуникации студентов и сотрудников СКФЭТ. Система построена на микросервисной архитектуре с использованием брокера сообщений Kafka и кэширования Redis, что обеспечивает высокую надёжность и производительность. Включает полноценный мессенджер с каналами, чатами в реальном времени и гибкой ролевой системой.

## Ролевая модель

Система поддерживает четыре роли пользователей с разграничением прав:

| Роль | Описание | Основные возможности |
|------|----------|---------------------|
| **Администратор** | Управление системой | Полный доступ, модерация пользователей, все каналы |
| **Преподаватель** | Учебный процесс | Создание уведомлений, управление каналами, приглашение участников |
| **Студент** | Основной пользователь | Получение уведомлений, участие в каналах, общение в чатах |
| **Родитель** | Контроль успеваемости | Просмотр уведомлений, ограниченное участие в каналах |

## Мессенджер и каналы

Встроенная система коммуникации с поддержкой реального времени:

### Каналы
- Создание тематических каналов для группового общения
- Пагинация и фильтрация списка каналов
- Поиск по названию

### Участники и роли канала
- Иерархия ролей: **Владелец → Администратор → Модератор → Участник**
- Приглашение пользователей с фильтрацией по ролям и группам
- Передача прав владельца (автоматическое понижение до администратора)
- Изменение ролей и удаление участников

| Роль в канале | Управление участниками | Удаление чужих сообщений |
|--------------|----------------------|-------------------------|
| Владелец | Полный доступ | Любые сообщения |
| Администратор | Модераторы и участники | Модераторы и участники |
| Модератор | Нет | Только участники |
| Участник | Нет | Нет |

### Чат в реальном времени
- Мгновенная доставка сообщений через SignalR
- Отправка текста и изображений
- Ответы на сообщения (reply)
- Редактирование собственных сообщений
- Статусы: доставлено (✓) и прочитано (✓✓)
- Индикатор печати (typing indicator)
- Пагинация с подгрузкой истории при скролле
- Группировка сообщений по датам и отправителям

### Приглашения
- Входящие и исходящие приглашения с пагинацией
- Принять / отклонить / отменить в один клик
- Удаление обработанных приглашений
- Баннер с количеством новых приглашений

## Telegram Боты — Триединая система регистрации

Система включает **трёх Telegram-ботов** с чётким разделением ответственности:

### 1. Бот для регистрации студентов
**Назначение:** Самостоятельная регистрация студентов в системе

| Функция | Описание |
|---------|----------|
| **Регистрация** | Студент вводит ФИО, группу, логин, пароль|
| **Валидация** | Проверка на уникальность аккаунта (один студент = один аккаунт) |
| **Статус** | После регистрации заявка уходит в Admin Bot на модерацию |
| **Уведомление** | При успешной модерации студент получает сообщение со ссылкой на APK |

### 2. Бот для регистрации сотрудников
**Назначение:** Регистрация учителей и администраторов

| Функция | Описание |
|---------|----------|
| **Регистрация** | Ввод ФИО, должности, логин, пароль |
| **Лимитирование** | Не более 3 регистраций в день с одного Telegram-аккаунта |
| **Проверка** | Предотвращение спам-регистраций |
| **Модерация** | Заявки направляются в Admin Bot |

### 3. Административный бот
**Назначение:** Управление пользователями и группами

| Функция | Описание |
|---------|----------|
| **Авторизация** | Вход по паролю для доступа к функциям |
| **Модерация заявок** | Просмотр входящих заявок на регистрацию |
| **Управление заявками** | Принять или отклонить заявку студента/сотрудника |
| **Управление группами** | Создание, редактирование, удаление учебных групп |
| **Рассылка APK** | Автоматическая отправка ссылки на Google Drive после подтверждения |

## Технологический стек

### Backend Core
- **.NET 9** — основная платформа
- **ASP.NET Core Web API** — RESTful API
- **Entity Framework Core** — ORM для PostgreSQL
- **SignalR** — real-time уведомления для мобильных клиентов
- **Clean Architecture** — разделение ответственности
- **JWT + Refresh Tokens** — безопасная аутентификация

### Инфраструктура
- **PostgreSQL 17.5** — основная база данных
- **Apache Kafka** — брокер сообщений для событийной коммуникации
- **Redis** — кэширование
- **Docker & Docker Compose** — контейнеризация и оркестрация

### Интеграции
- **Telegram.Bot** — библиотека для работы с Telegram API
- **Confluent.Kafka** — Kafka клиент для .NET
- **StackExchange.Redis** — Redis клиент

### Бэкап и восстановление
- Автоматический бэкап PostgreSQL — ежедневно в 03:00
- Через запрос api/backup/create
